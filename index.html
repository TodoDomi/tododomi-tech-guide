React Native tododomi mobile apps

Apps

Customer

This is the app for the final consumer, the biggest and most complex one.
The core flow of the service starts in this application. A user can signup, login, search for stores and items and add them to a cart. Finally it can make a pruchase selecting a payment method, credit cards are suported.

The app has many features including but not limited to:

  • Select replacement items in case stock problems.
  • Add special instructions for each product that shopper can read.
  • Add special instrutions for the delivery.
  • Customize some products that support it. For example adding drinks and side dishes.
  • Filter and sorting shops.
  • Schedule future orders.
  • Manage delivery addresses.
  • Manage payment methods.
  • Rate shops and past orders.
  • Give tips.
  • View past orders.

Shopper

The shopper is one of the apps for tododomi workers. Shoppers are the first to handle user orders. They are suposed to be in shops and buy items for each user purchase. The core flow of the shopper app is to accept an order, buy the items and mark a purchase as finished adn ready to be picked up.

Some extra features:

  • Change items for the preselected replacements.
  • Ability to call the user.
  • Change final order price and add a reason and a photo of the reciept.

Restaurant

Similarly to the shopper, the restaurant manager accepts orders, selects a preparation time and finally marks them as ready to deliver.

Features:

  • See selected options for each meal.
  • Close the reception of new orders.
  • Reject an order.

Delivery

The delivery app is the last one in the chain. Deliverys pick up orders from restaurants/shops and deliver them to final customers. They charge the user in case the payment method is cash.

Features:

  • See optimzed delivery routes
  • Open google maps with marked destinations
  • Mark orders as not delivered
  • Call the user

React-Redux

The four applications use:

  • react-native and redux as the state management solution.
  • react-native-navigation as the navigation library.
  • redux-thunks to handle async tasks on redux actions.

All the apps use immutable as a way of ensuring immutability on redux reducers, with the exception of the restaurant app that uses redux-starter-kit which includes immer as the immutability library.

All the backend endpoints that the app uses are delared under src/api.
This services are then used in thunks under src/actions

The 3 apps using immutable make extensive use of selectors (src/selectors) in order to avoid recopmutation of the expensive toJS method of immutable objects.

react-native-navigation screens are declared in src/screens
There is a container (src/containers) for each of these screens.

Sessions are managed by redux-react-native-session and persisted to local-storage. The authentication method is token based and is addapted to the default method in the ruby gem devise-token-auth
